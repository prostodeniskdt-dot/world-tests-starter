# О том О сем

Проект: **образовательная платформа для барменов и HoReCa** — тесты, рейтинг, сочетания вкусов, база знаний, каталоги алкоголя, коктейлей, техники и посуды. **Next.js** + **PostgreSQL** (Timeweb Cloud, локальный PostgreSQL).

## Что уже есть
- 🔐 **Система аутентификации с JWT токенами** — безопасная регистрация и вход с использованием bcrypt для хеширования паролей и JWT токенов в httpOnly cookies
- 🔑 **Восстановление пароля** — функционал сброса пароля через email с токенами
- 🎨 **Современный UI входа** — полноэкранная страница авторизации в стиле BAR BOSS ONLINE с визуализацией статистики
- 📝 **Система управления тестами** — динамическая загрузка тестов из базы данных
- 🏆 Страница `/leaderboard` — общий рейтинг (по сумме очков) с пагинацией, именами, фамилиями и кликабельными Telegram аккаунтами
- 👤 Страница `/profile` — личный кабинет пользователя с историей пройденных тестов (автоматическое определение пользователя из JWT)
- 📊 **Живой рейтинг** — обновление рейтинга в реальном времени на главной странице
- 🛡️ **Rate Limiting** — защита API от злоупотреблений
- 🚫 **Базовая защита от читов** — проверка времени прохождения и частоты попыток
- ⚠️ **Обработка ошибок** — Error Boundary и централизованная обработка ошибок
- API:
  - `POST /api/auth/register` — регистрация нового пользователя (с паролем)
  - `POST /api/auth/login` — вход пользователя (email + пароль)
  - `GET /api/auth/verify` — проверка JWT токена
  - `POST /api/auth/logout` — выход пользователя
  - `POST /api/auth/forgot-password` — запрос на сброс пароля
  - `POST /api/auth/reset-password` — сброс пароля по токену
  - `POST /api/auth/check-email` — проверка существования email
  - `GET /api/tests` — список активных тестов
  - `GET /api/tests/[testId]` — получение теста с вопросами
  - `GET /api/tests/[testId]/attempts` — информация о попытках пользователя
  - `POST /api/submit` — отправка результата теста (начисление очков)
  - `GET /api/leaderboard` — рейтинг с пагинацией (JSON)
  - `GET /api/user/profile` — профиль пользователя
  - `GET /api/user/attempts` — история попыток пользователя

> Подключение к БД — через **DATABASE_URL** (драйвер `pg`). Используется только на сервере (API, Server Components).

## Быстрый старт локально

### 1) Установить зависимости
```bash
npm i
```

### 2) База данных PostgreSQL
Создай БД (Timeweb Cloud, локальный PostgreSQL 18 и т.д.) и выполни инициализацию схемы. Консольные команды для миграций — в файле **MIGRATION_COMMANDS.md**.

Кратко: выполни `db/init.sql` (полная инициализация), затем при необходимости миграции из `db/migrations/` по порядку.

### 3) Настроить переменные окружения
Создай файл `.env.local` в корне проекта:

```bash
# PostgreSQL (обязательно)
DATABASE_URL=postgresql://USER:PASSWORD@HOST:PORT/DATABASE

# Для облачной БД (например Timeweb) включи SSL:
# DB_SSL=true

# JWT
JWT_SECRET=your-super-secret-key-min-32-chars-change-in-production
JWT_EXPIRES_IN=7d

# URL приложения (для ссылок сброса пароля)
NEXT_PUBLIC_APP_URL=http://localhost:3000

# Email (опционально, для сброса пароля)
# SMTP_HOST=smtp.example.com
# SMTP_PORT=587
# SMTP_USER=your-email@example.com
# SMTP_PASS=your-password
```

**Важно:**
- `JWT_SECRET` должен быть минимум 32 символа в продакшене
- `NEXT_PUBLIC_APP_URL` используется для генерации ссылок восстановления пароля
- Email настройки опциональны (в разработке ссылки выводятся в консоль)

### 4) Запуск
```bash
npm run dev
```

Открой: http://localhost:3000

В настройках окружения задай переменные:
- `DATABASE_URL` — строка подключения к PostgreSQL

- `JWT_EXPIRES_IN` (опционально, по умолчанию 7d)

## Функционал аутентификации

### Регистрация и вход

При первом посещении сайта пользователь видит полноэкранную страницу входа с:
- **Левая часть**: визуализация статистики участников
- **Правая часть**: форма входа/регистрации с переключением табов

**Форма регистрации:**
- **Email** (обязательно, уникальный, проверка в реальном времени)
- **Имя** (обязательно, 1-50 символов)
- **Фамилия** (обязательно, 1-50 символов)
- **Пароль** (обязательно, минимум 8 символов, заглавная и строчная буква, цифра)
- **Подтверждение пароля** (обязательно, должно совпадать)
- **Ник в Telegram** (необязательно, для связи с участниками)

**Форма входа:**
- **Email** (обязательно)
- **Пароль** (обязательно, с возможностью показать/скрыть)
- Ссылка "Забыли пароль?" (готово к реализации)

**Требования к паролю:**
- Минимум 8 символов
- Хотя бы одна заглавная буква (A-Z)
- Хотя бы одна строчная буква (a-z)
- Хотя бы одна цифра (0-9)

Пароли хранятся в БД в виде bcrypt хешей (10 раундов соли).

После успешной регистрации или входа пользователь попадает на главную страницу с:
- Живым рейтингом слева (обновляется каждые 10 секунд)
- Доступными тестами справа
- Иконкой личного кабинета в верхнем правом углу

В рейтинге отображается:
- Имя и фамилия
- Email
- **Telegram ник (кликабельный)** — ссылка на `https://t.me/@username` для связи с участником

## Безопасность

- Пароли хешируются с использованием bcrypt (10 раундов соли)
- JWT токены хранятся в httpOnly cookies (защита от XSS)
- Подключение к БД только на сервере (API, Server Components)
- Валидация паролей на клиенте и сервере
- Защита от SQL инъекций через параметризованные запросы (pg)
- Rate Limiting на критичных API эндпоинтах
- Базовая защита от читов (проверка времени прохождения, лимиты попыток)

## Миграция существующих пользователей

Если у вас уже есть пользователи без паролей, они не смогут войти в систему. Для миграции:
1. Создайте скрипт для установки временных паролей
2. Отправьте пользователям инструкции по восстановлению пароля
3. Или реализуйте функционал восстановления пароля
