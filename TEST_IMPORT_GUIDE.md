# Импорт тестов из JSON и Markdown

Эта инструкция описывает фактически поддерживаемый формат импорта тестов в систему.

Импорт доступен администратору на странице `/admin/tests/import`. Можно:

1. вставить содержимое файла в поле;
2. загрузить файл `.json`, `.md` или `.txt`;
3. нажать **«Preview / Валидировать»**;
4. исправить все ошибки;
5. нажать **«Импортировать черновик»**.

Импортированный тест сохраняется как **черновик** и не публикуется автоматически.

## Важные правила

- Рекомендуемый формат — JSON.
- Кодировка файла — UTF-8.
- Все индексы ответов начинаются с `0`, а не с `1`.
- Каждый вопрос должен иметь уникальный `id`.
- Для каждого вопроса должен существовать ключ с тем же `id` в `answerKey`.
- Не добавляйте в `answerKey` ключи, которых нет среди вопросов.
- Максимум 100 вопросов в одном тесте.
- Максимум 20 вариантов, строк, столбцов или пропусков в одной механике.
- Поддерживаются только механики, перечисленные ниже.
- Перед импортом всегда используйте preview: он показывает точный путь к ошибке.

## Общая структура JSON

```json
{
  "schemaVersion": 1,
  "id": "bartender-basics",
  "title": "Основы работы бармена",
  "description": "Проверка базовых знаний",
  "category": "Бар",
  "author": "Иван Иванов",
  "difficultyLevel": 1,
  "basePoints": 200,
  "maxAttempts": null,
  "questions": [],
  "answerKey": {}
}
```

### Поля теста

- `schemaVersion` — версия схемы. Сейчас допустимо только число `1`.
- `id` — необязательный идентификатор теста: 3–64 символа, только строчные латинские буквы, цифры и дефисы. Первый символ — буква или цифра.
- `title` — обязательное название, от 1 до 200 символов.
- `description` — необязательное описание, до 2000 символов.
- `category` — необязательная категория, до 100 символов.
- `author` — необязательный автор, до 120 символов.
- `difficultyLevel` — сложность: `1`, `2` или `3`.
- `basePoints` — базовые баллы: целое число от 50 до 1000.
- `maxAttempts` — лимит попыток: положительное целое число или `null` без ограничения.
- `questions` — массив от 1 до 100 вопросов.
- `answerKey` — объект с правильными ответами. Имя каждого ключа совпадает с `id` вопроса.

Необязательные поля можно пропустить. Система подставит значения по умолчанию:

```json
{
  "schemaVersion": 1,
  "description": "",
  "category": "",
  "author": "",
  "difficultyLevel": 1,
  "basePoints": 200,
  "maxAttempts": null
}
```

## Общие поля вопроса

Каждый вопрос содержит:

```json
{
  "id": "q1",
  "type": "multiple-choice",
  "text": "Текст вопроса",
  "hint": "Необязательная подсказка",
  "media": {
    "type": "image",
    "url": "https://example.com/image.jpg",
    "alt": "Описание изображения",
    "caption": "Необязательная подпись"
  }
}
```

- `id` — обязательная уникальная строка длиной до 64 символов.
- `type` — обязательный тип механики.
- `text` — текст вопроса, до 5000 символов.
- `hint` — необязательная подсказка, до 8000 символов.
- `media` — необязательное изображение.
- `media.url` — обязательный абсолютный URL изображения, если указан `media`.
- `media.alt` — альтернативный текст до 300 символов.
- `media.caption` — подпись до 500 символов.
- `imageUrl` — старое совместимое поле URL изображения. Для новых файлов используйте `media`.
- `videoUrl` — необязательный URL видео.

## Подсказки и объяснения после ответа

Чтобы у вопроса отображалась подсказка, добавьте строковое поле `hint` **внутрь самого вопроса**:

```json
{
  "id": "q1",
  "type": "multiple-choice",
  "text": "Для чего используется джиггер?",
  "hint": "Джиггер — мерный барный инструмент для точного дозирования ингредиентов.",
  "options": [
    "Для измерения объёма",
    "Для разминания фруктов",
    "Для фильтрации напитка"
  ]
}
```

В `answerKey` подсказку добавлять не нужно:

```json
{
  "answerKey": {
    "q1": 0
  }
}
```

Поле `hint`:

- необязательное;
- поддерживается у всех девяти типов вопросов;
- может содержать до 8000 символов;
- должно находиться рядом с `id`, `type` и `text` внутри объекта вопроса;
- не показывается участнику до завершения теста;
- после отправки теста отображается как блок **«Справка»** вместе с результатом ответа.

В административном интерактивном preview подсказка сейчас скрыта, потому что эта страница показывает вопрос до проверки ответа. Проверить наличие текста можно в редакторе теста, а его отображение участнику — после отправки теста.

Неверно — `hint` находится снаружи вопроса:

```json
{
  "questions": [
    {
      "id": "q1",
      "type": "multiple-choice",
      "text": "Вопрос",
      "options": ["A", "B"]
    }
  ],
  "hint": "Этот текст не относится к вопросу",
  "answerKey": {
    "q1": 0
  }
}
```

Правильно:

```json
{
  "questions": [
    {
      "id": "q1",
      "type": "multiple-choice",
      "text": "Вопрос",
      "hint": "Объяснение правильного ответа",
      "options": ["A", "B"]
    }
  ],
  "answerKey": {
    "q1": 0
  }
}
```

Пример старого совместимого поля:

```json
{
  "id": "q1",
  "type": "multiple-choice",
  "text": "Что изображено?",
  "imageUrl": "https://example.com/old-image.jpg",
  "options": ["Шейкер", "Стрейнер"]
}
```

## Поддерживаемые механики

### 1. Один вариант — `multiple-choice`

```json
{
  "questions": [
    {
      "id": "q1",
      "type": "multiple-choice",
      "text": "Какой инструмент используют для процеживания?",
      "hint": "Стрейнер удерживает лёд и крупные частицы при переливании напитка.",
      "options": ["Стрейнер", "Мадлер", "Джиггер"]
    }
  ],
  "answerKey": {
    "q1": 0
  }
}
```

`answerKey.q1` — индекс одного правильного варианта. Здесь `0` означает первый вариант — «Стрейнер».

### 2. Несколько вариантов — `multiple-select`

```json
{
  "questions": [
    {
      "id": "q2",
      "type": "multiple-select",
      "text": "Выберите цитрусовые",
      "hint": "Лимон, апельсин и лайм относятся к цитрусовым.",
      "instruction": "Можно выбрать несколько ответов",
      "options": ["Лимон", "Апельсин", "Яблоко", "Лайм"]
    }
  ],
  "answerKey": {
    "q2": [0, 1, 3]
  }
}
```

Правильный ответ — массив индексов всех правильных вариантов.

### 3. Верно/неверно с причиной — `true-false-enhanced`

```json
{
  "questions": [
    {
      "id": "q3",
      "type": "true-false-enhanced",
      "text": "Оцените утверждение",
      "hint": "Джиггер позволяет точно отмерять объём ингредиентов.",
      "statement": "Джиггер используется для измерения объёма.",
      "reasons": [
        "Это мерный инструмент",
        "Это инструмент для фильтрации"
      ]
    }
  ],
  "answerKey": {
    "q3": {
      "answer": true,
      "reason": 0
    }
  }
}
```

- `answer` — `true` или `false`.
- `reason` — индекс правильной причины.
- Если массив `reasons` непустой, `reason` обязателен.

### 4. Пропуски с выпадающими списками — `cloze-dropdown`

В тексте используйте обозначения `[1]`, `[2]` и далее. Массив `gaps` должен идти в том же порядке.

```json
{
  "questions": [
    {
      "id": "q4",
      "type": "cloze-dropdown",
      "text": "Для сауэра обычно используют [1], кислую часть и [2].",
      "hint": "Классическая формула сауэра включает крепкую, кислую и сладкую части.",
      "gaps": [
        {
          "index": 0,
          "options": ["крепкую основу", "газированную воду"]
        },
        {
          "index": 1,
          "options": ["сладкую часть", "соль"]
        }
      ]
    }
  ],
  "answerKey": {
    "q4": [0, 0]
  }
}
```

Ответ — массив индексов правильных вариантов: по одному числу для каждого пропуска.

### 5. Поиск ошибок — `select-errors`

Самый простой способ — отметить выбираемые фрагменты двойными квадратными скобками. Во время импорта скобки будут удалены, а `markedParts` сформируется автоматически.

```json
{
  "questions": [
    {
      "id": "q5",
      "type": "select-errors",
      "text": "Найдите ошибки",
      "hint": "Мадлер разминает ингредиенты, джиггер измеряет объём.",
      "content": "Для измерения объёма используют [[мадлер]], а лёд процеживают через [[джиггер]].",
      "allowMultiple": true
    }
  ],
  "answerKey": {
    "q5": [1, 2]
  }
}
```

Важно: для этой механики ответы — **ID фрагментов**, начинающиеся с `1`, а не индексы с `0`.

Можно задать фрагменты вручную:

```json
{
  "id": "q5",
  "type": "select-errors",
  "text": "Найдите ошибку",
  "content": "Лёд процеживают через джиггер.",
  "markedParts": [
    {
      "id": 1,
      "text": "джиггер",
      "start": 22,
      "end": 29
    }
  ],
  "allowMultiple": false
}
```

`start` и `end` необязательны. Рекомендуется использовать разметку `[[...]]`, чтобы не вычислять позиции вручную.

### 6. Сопоставление — `matching`

```json
{
  "questions": [
    {
      "id": "q6",
      "type": "matching",
      "text": "Сопоставьте инструмент и назначение",
      "hint": "Джиггер измеряет, мадлер разминает, стрейнер фильтрует.",
      "variant": "1-to-1",
      "leftItems": ["Джиггер", "Мадлер", "Стрейнер"],
      "rightItems": ["Измерение", "Разминание", "Фильтрация"]
    }
  ],
  "answerKey": {
    "q6": [[0, 0], [1, 1], [2, 2]]
  }
}
```

Каждая пара имеет формат `[индекс_слева, индекс_справа]`. Все индексы начинаются с `0`. Число пар должно совпадать с числом элементов `leftItems`.

### 7. Упорядочивание — `ordering`

```json
{
  "questions": [
    {
      "id": "q7",
      "type": "ordering",
      "text": "Расположите действия по порядку",
      "hint": "Сначала подготовьте рабочее место, затем готовьте и подавайте напиток.",
      "instruction": "Начните с подготовки рабочего места",
      "items": [
        "Подготовить инструменты",
        "Отмерить ингредиенты",
        "Смешать",
        "Подать"
      ]
    }
  ],
  "answerKey": {
    "q7": [0, 1, 2, 3]
  }
}
```

Ответ — массив индексов элементов в правильном порядке. Длина массива должна совпадать с длиной `items`.

Например, порядок «второй, первый, третий» записывается как `[1, 0, 2]`.

### 8. Два последовательных шага — `two-step`

```json
{
  "questions": [
    {
      "id": "q8",
      "type": "two-step",
      "text": "Выберите основу и подходящий способ приготовления",
      "hint": "Способ приготовления зависит от состава и желаемой текстуры напитка.",
      "step1": {
        "question": "Какая основа используется?",
        "options": ["Джин", "Ром"]
      },
      "step2": {
        "question": "Какой способ подходит?",
        "options": ["Шейк", "Стир"]
      }
    }
  ],
  "answerKey": {
    "q8": {
      "step1": 0,
      "step2Mapping": {
        "0": 1,
        "1": 0
      }
    }
  }
}
```

- `step1` — индекс правильного ответа первого шага.
- `step2Mapping` — соответствие каждого индекса первого шага правильному индексу второго шага.
- Ключи `step2Mapping` в JSON всегда строки: `"0"`, `"1"` и так далее.

### 9. Матрица — `matrix`

#### Один ответ в каждой строке

```json
{
  "questions": [
    {
      "id": "q9",
      "type": "matrix",
      "text": "Выберите категорию для каждого напитка",
      "hint": "Мартини обычно перемешивают, а Дайкири встряхивают.",
      "rows": ["Мартини", "Дайкири"],
      "columns": ["Стир", "Шейк"],
      "matrixType": "single-select"
    }
  ],
  "answerKey": {
    "q9": {
      "0": 0,
      "1": 1
    }
  }
}
```

Ключ объекта — индекс строки, значение — индекс правильного столбца.

#### Несколько ответов в каждой строке

```json
{
  "questions": [
    {
      "id": "q10",
      "type": "matrix",
      "text": "Выберите характеристики",
      "hint": "Лимон кислый и цитрусовый, сироп — сладкий.",
      "rows": ["Лимон", "Сироп"],
      "columns": ["Кислый", "Сладкий", "Цитрусовый"],
      "matrixType": "multiple-select"
    }
  ],
  "answerKey": {
    "q10": {
      "0": [0, 2],
      "1": [1]
    }
  }
}
```

Значение каждой строки — массив индексов правильных столбцов.

```json
  "description": "Минимальный корректный файл",

```

## Формат Markdown

Markdown-файл — это оболочка из простого YAML frontmatter и обязательного JSON-блока.

Файл должен:

1. начинаться с `---`;
2. содержать метаданные в формате `ключ: значение`;
3. закрывать метаданные вторым `---`;
4. содержать fenced-блок с точной меткой `json`;
5. хранить `questions` и `answerKey` внутри JSON-блока.

Пример `example-test.md`:

````markdown
---
schemaVersion: 1
title: Основы работы с инвентарём
description: Короткий учебный тест
category: Инвентарь
author: Администратор
difficultyLevel: 1
basePoints: 200
maxAttempts: null
---

# Основы работы с инвентарём

Текст между frontmatter и JSON-блоком можно использовать как комментарий для человека.
Он не преобразуется в вопросы.

```json
{
  "questions": [
    {
      "id": "q1",
      "type": "multiple-choice",
      "text": "Для чего нужен джиггер?",
      "hint": "Джиггер нужен для точного измерения объёма ингредиентов.",
      "options": [
        "Для измерения объёма",
        "Для разминания фруктов",
        "Для фильтрации"
      ]
    },
    {
      "id": "q2",
      "type": "multiple-select",
      "text": "Выберите цитрусовые",
      "hint": "Лимон и лайм относятся к цитрусовым.",
      "options": ["Лимон", "Лайм", "Яблоко"]
    }
  ],
  "answerKey": {
    "q1": 0,
    "q2": [0, 1]
  }
}
```
````

### Ограничения Markdown

- Это не свободный текстовый формат.
- Заголовки, списки и чекбоксы Markdown не превращаются в вопросы.
- Вопросы и ответы обязательно задаются валидным JSON внутри блока ` ```json `.
- Парсер frontmatter поддерживает простые строки, положительные целые числа и `null`.
- Не используйте вложенный YAML, YAML-массивы или многострочные YAML-значения.
- Если поле присутствует и во frontmatter, и в JSON-блоке, значение frontmatter имеет приоритет.
- Поле `id` теста из JSON-блока в текущем Markdown-импорте не используется. Если нужен контролируемый `id`, импортируйте чистый JSON.

## Legacy-типы

Система умеет преобразовывать старые типы:

- `best-example` → `multiple-choice`;
- `scenario` → `multiple-choice`, `ordering` или `matching` по `actionType`;
- `construct` → `ordering` или `multiple-select`;
- `grouping` → `matching`.

При preview такие преобразования показываются как предупреждения. Для новых файлов не используйте legacy-типы — сразу выбирайте одну из девяти поддерживаемых механик.

## Типичные ошибки

### Индексы записаны с единицы

Неверно:

```json
{
  "options": ["A", "B", "C"],
  "answerKey": { "q1": 1 }
}
```

Если правильный вариант — первый, должно быть:

```json
{
  "answerKey": { "q1": 0 }
}
```

Исключение — `select-errors`: там используются ID фрагментов, обычно `1`, `2`, `3`.

### `id` вопроса не совпадает с ключом ответа

Неверно:

```json
{
  "questions": [{ "id": "q1" }],
  "answerKey": { "question1": 0 }
}
```

Правильно:

```json
{
  "questions": [{ "id": "q1" }],
  "answerKey": { "q1": 0 }
}
```

### JSON содержит комментарии или лишнюю запятую

JSON не поддерживает комментарии и запятую после последнего элемента.

Неверно:

```json
{
  "title": "Тест",
  "difficultyLevel": 1,
}
```

### Неправильные кавычки

Используйте двойные прямые кавычки `"`, а не одинарные `'` и не типографские `«»`.

### Пустой тест

В `questions` должен быть хотя бы один вопрос, и у него должен быть правильный ответ в `answerKey`.

### Неполный `matching` или `ordering`

- В `matching` число пар должно совпадать с числом элементов слева.
- В `ordering` длина ответа должна совпадать с числом элементов.
- Все индексы должны находиться в допустимом диапазоне.

## Проверка перед импортом

Проверьте файл по списку:

- [ ] файл сохранён в UTF-8;
- [ ] JSON синтаксически корректен;
- [ ] `schemaVersion` равен `1`;
- [ ] `title` заполнен;
- [ ] `questions` содержит хотя бы один вопрос;
- [ ] все `id` вопросов уникальны;
- [ ] для каждого `id` есть ключ в `answerKey`;
- [ ] у каждого вопроса, для которого нужна справка после ответа, поле `hint` находится внутри объекта вопроса;
